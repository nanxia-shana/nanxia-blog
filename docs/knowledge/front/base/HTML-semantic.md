# HTML 标签：从基础到语义化布局

## 📚 一、文档结构与元数据标签
这类标签定义了文档的整体结构、字符编码、标题、引入外部资源等，对浏览器和搜索引擎至关重要。
|标签|名称|常用场景|特别注意|
|---|---|---|---|
|\<!DOCTYPE html>|文档类型声明|每个 HTML5 文档的开头第一行。|必须放在最顶部，告诉浏览器使用 HTML5 标准解析。|
|\<html>         |HTML 根元素|包裹整个 HTML 文档。         |通常包含 lang属性，如 \<html lang="zh-CN">，用于声明页面主要语言，利于 SEO 和无障碍访问。|
|\<head>          |头部|包含页面的元数据（metadata），如标题、字符集、CSS/JS 链接。对用户不可见。|内部不直接放置可见内容。|
|\<meta charset="UTF-8">|字符集声明|在 \<head> 内，指定文档使用 UTF-8 编码。|强烈建议始终使用，以避免中文等非英文字符出现乱码。|
|\<meta name="viewport" ...>|视口设置|在 \<head> 内，控制移动端浏览器的视口大小和缩放。|响应式设计的必备标签，否则手机上页面会显示过大。|
|\<title>|页面标题|在 \<head> 内，定义浏览器标签页上显示的标题。|极其重要的 SEO 因素，应简洁明了地概括页面内容。|
|\<link rel="stylesheet" href="...">|外部样式表|在 \<head> 内，引入外部 CSS 文件。|rel属性必须设为 "stylesheet"。href指向 CSS 文件路径。|
|\<style>|内部样式表|在 \<head>或 \<body>内，编写页面内的 CSS 样式。|通常用于小型项目或单个组件的样式。大型项目建议使用外部 CSS。|
|\<script>|JavaScript 脚本|在 \<head>或 \<body>结束前，引入或编写 JS 代码。|- 性能考量：放在 \<body>结束前可避免阻塞页面渲染。<br> - async/defer：用于异步加载脚本，优化性能。<br> - src：用于引入外部 JS 文件。|

## 📰 二、内容分区与语义化标签 (HTML5 核心)

HTML5 引入了一系列语义化标签，用以清晰地描述内容的结构和含义，取代了过去滥用 `<div>` 的做法。

|标签|名称|常用场景|特别注意
|---|---|---|---|
|\<header>|页眉|页面或区块的头部，通常包含 logo、主导航、搜索框。|一个页面可以有多个 \<header>（如页面头部和文章头部）。
|\<nav>|导航|包含主要的导航链接区域。|并非所有链接组都用 \<nav>，它特指主要导航（如主导航菜单）。侧边栏链接、页脚链接群可用 \<div> 或 \<footer>。
|\<main>|主要内容|文档的主体内容，每个页面应有且只有一个​ \<main>。|它不包含侧边栏、导航、页脚等重复内容。屏幕阅读器用户可通过此标签快速跳转到核心内容。
|\<article>|文章|独立的、可自我包含的内容块，如博客文章、新闻报道、论坛帖子。|内容在被移除出上下文后依然有意义。
|\<section>|章节|文档中的一个主题性分组，通常有标题。|用于对内容进行分块，比 \<div> 更有语义。不要为了样式而用它代替 \<div>。
|\<aside>|侧边栏|与周围内容间接相关的内容，如广告、引述、相关链接。|可以理解为“旁白”或“附加信息”。
|\<footer>|页脚|页面或区块的底部，通常包含版权信息、联系方式、相关链接。|一个页面可以有多个 \<footer>。
|\<div>|通用容器|当没有其他更合适的语义标签时，用作纯粹的样式或脚本钩子。|慎用：优先考虑语义化标签。过度使用会导致“Div Soup”，不利于可访问性和SEO。

## ✍️ 三、文本内容标签
用于组织和展示文本、标题、列表等内容。
|标签|名称|常用场景|特别注意
|---|---|---|---|
|\<h1> ~ \<h6>|标题|定义从最重要到最不重要的六级标题。|一个页面通常只有一个 \<h1>，用于页面主标题。层级关系必须正确（h1 -> h2 -> h3...），不能跳级。
|\<p>|段落|定义文本段落。|浏览器会自动在段落前后添加空白。不要用 \<br> 来换行创建新段落。
|\<span>|行内容器|用于对文本的一小部分进行样式化或脚本操作。|本身无语义，是 \<div> 的行内版本。
|\<br>|换行|在文本中强制插入一个换行符。|避免在段落内用多个 \<br> 来调整间距，这是错误的做法。应使用 CSS 的 margin或 padding。
|\<hr>|水平分割线|表示内容主题的重大转变（如话题转换）。|不只是用来画一条线，它有语义含义。
|列表​
|\<ul>|无序列表|列表项没有特定顺序，通常用项目符号表示。|内部只能直接包含 \<li>。
|\<ol>|有序列表|列表项有特定顺序（如步骤1,2,3），通常用数字表示。|内部只能直接包含 \<li>。\<ol> 有 start和 reversed属性。
|\<li>|列表项|用于 \<ul> 或 \<ol> 中。
|\<dl>|描述列表|包含一组术语及其描述的列表。|内部由 \<dt>(术语) 和 \<dd>(描述) 组成。
|文本格式化​
|\<strong>|重要性强调|表示内容非常重要，通常以粗体显示。|语义化标签，强调重要性。与 \<b> (仅样式加粗) 不同。
|\<em>|语气强调|表示内容的语气强调，通常以斜体显示。|语义化标签，改变句子含义。与 \<i> (仅样式斜体) 不同。
|\<mark>|标记/高亮|表示被标记的或高亮的文本。|用于突出显示与用户当前活动相关的文本。
|\<code>|代码片段|表示计算机代码。|通常配合 \<pre> 使用来保留空格和换行。
|\<pre>|预格式化文本|保留文本原有的空格和换行符。|常用于显示代码或诗歌。

## 🧱 四、媒体与嵌入内容标签
用于在网页中嵌入图片、音视频、其他网页等资源。
|标签|名称|常用场景|特别注意
|---|---|---|---|
|\<img>|图像|嵌入图片。|- 必须包含 alt属性，用于图片无法显示时的替代文本，对无障碍访问和 SEO​ 至关重要。<br> - src属性指定图片路径。- 考虑使用 loading="lazy"实现懒加载，优化性能。
|\<audio>|音频|嵌入音频内容。|- 使用 \<source> 标签提供多种格式以保证兼容性。- controls属性显示播放控件。- 提供 \<track> 添加字幕。
|\<video>|视频|嵌入视频内容。|同 \<audio>，使用 \<source> 和 controls。注意设置宽高。
|\<iframe>|内联框架|在当前页面嵌入另一个 HTML 页面。- 安全风险：可能引入第三方恶意内容。使用 sandbox属性进行安全限制。- 性能杀手：每个 iframe 都是一个独立的文档，会显著增加页面负载。
|\<svg>|SVG 矢量图|嵌入可缩放的矢量图形。|矢量图放大不失真，适合图标、图表。可以通过 CSS 和 JS 进行操作。
|\<canvas>|画布|通过 JavaScript 动态绘制图形、动画。|本身是一个位图容器，绘制的图形不是 DOM 的一部分，难以进行无障碍访问。

## 🔗 五、链接与交互标签
用于创建超链接和实现基础的用户交互。
|标签|名称|常用场景|特别注意
|---|---|---|---|
|\<a>|锚点/链接|创建超链接，跳转到其他页面、页面内锚点或下载文件。|- href属性不能为空，空值 (href="") 可能导致页面刷新。<br> - 使用 target="_blank"在新标签页打开时，务必加上 rel="noopener noreferrer"以防止安全漏洞。<br> - download属性可指示浏览器下载链接资源而非导航。
|\<button>|按钮|创建一个可点击的按钮。|- 首选用于表单提交或任何 JavaScript 交互。<br> - 比 \<input type="button"> 更灵活，可以包含 HTML 内容（如图标+文字）。<br> - 默认有 type="submit"，在表单内使用时需注意，可设 type="button"避免意外提交。

## 📝 六、表单标签
用于收集用户输入。
|标签|名称|常用场景|特别注意
|---|---|---|---|
|\<form>|表单|包裹一组输入控件，用于提交数据。|action属性定义数据提交到的 URL，method定义提交方法 (GET/POST)。
|\<input>|输入框|创建多种类型的输入字段。|- type属性决定其类型（text, email, password, number, date, checkbox, radio, file, submit...）。<br> - 必须为输入控件关联 \<label>，通过 id和 for属性绑定，极大提升可用性。
|\<label>|标签|为表单控件定义标签。|点击标签会聚焦到对应控件，对复选框和单选按钮尤其有用。
|\<textarea>|多行文本框|接收多行文本输入。|与 \<input> 不同，它有起始和结束标签，内容写在中间。
|\<select>|下拉列表|创建下拉选择框。|内部由 \<option> 组成。\<optgroup> 可对选项进行分组。
|\<button>|按钮|提交表单或执行其他操作。|在表单内，type="submit"是默认值。

## 💡 核心建议与最佳实践

1. **语义化优先：** 编写 HTML 时，首先考虑内容的含义（这是什么？），而不是外观（看起来怎么样？）。选择正确的标签能让浏览器、搜索引擎和辅助技术更好地理解你的页面。
2. **无障碍访问 (a11y)：** 很多标签和属性（如 alt, label, lang, 正确的标题层级）都是为了帮助残障人士使用网页。这是专业开发者的责任。
3. **SEO 友好：** 搜索引擎爬虫依赖 HTML 结构和语义来理解页面内容。语义化标签和有意义的标题/内容对排名有积极影响。
4. **避免过时/废弃标签：** 如 \<font>, \<center>, \<frame>，这些标签控制样式的做法已被 CSS 取代，且不利于维护和可访问性。
5. **\<div> 和 \<span> 是最后的选择：** 它们是强大的工具，但应在没有更合适语义标签时才使用。
